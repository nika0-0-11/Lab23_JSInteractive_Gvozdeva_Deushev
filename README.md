# Крестики нолики (Tic-tac-toe)

## Описание игры:

### Правила

- Игроки по очереди ставят на свободные клетки поля 3x3 своей знак: перый игрок `X`, второй - `0`.
- Побеждает тот, кто первый выстроит три своих знака в ряд (по горизонтали, вертикали или диагонали).
- Если все клетки заполнены, а победителя нет, объявляется ничья.

### Логика победы/поражения

- Игра проверяет состояние после каждого хода.
- При обнаружении выиграшной комбиннации игра завершается, победитель отображается в статусе.
- При заполнении всех клеток без выигрышной комбинации фиксируется ничья.

### Особенности реализации

- Игра полностью написана на JavaScript, HTML и CSS.
- После завершения партии новые клики по клеткам игнорируются.
- Кнопка `Начать заново` полностью сбрасывает состояние игры.

## Игровая механика:

### Как определяется победитель?

- В коде хранятся все 8 возможных выигрышных комбинаций.
- После каждого хода проверяется, есть ли комбинация, где все три клетки заняты одним и тем же текущим символом.

### Как обрабатываются клики?

- Каждой клетке назначается обработчик события `click`.
- При клике проверяется: активна ли игра, пуста ли клетка.
- Если условия соблюдены - в клетку ставится текущий символ, обновляется массив `board`, затем проверяется победа/ничья или происходит смена игрока.

### Как работает перезапуск?

- Сбрасывается массив `board`.
- Текущий игрок становится `X`.
- Флаг `gameActive` устанавливается в `true`.
- Очищается визуальное отображение всех клеток.
- Обновляется текст статуса.

## Структура данных:

### Массив `board`:

- Представляет собой массив из 9 строк.
- Каждый элемент соответствует клетке игрового поля.
- Пустая клетка хранит `" "`, занятая - `X` или `0`.

### Условия победы

- Содержит 8 массивов с индексами клеток, образующих вигрышные линии:
    - 3 горизонталии: `[0, 1, 2]`, `[3, 4, 5]`, `[6, 7, 8]`
    - 3 вертикали: `[0, 3, 6]`, `[1, 4, 7]`, `[2, 5, 8]`
    - 2 диагонали: `[0, 4, 8]`, `[2, 4, 6]`

### Хранение состояние игры:

- `currentPlayer` - кто ходит сейчас.
- `gameActive` - идет ли игра.
- `board` - текущее состояние поля.

## Алгоритмы

### Алгоритм проверки победы

1. Перебираем все выиграшные крмбинации из `winConditions`.
2. Для каждой комбинаци получаем символы из `board` по трем индексам.
3. Если все три симвлда не пустые и одинаковые - объявляем победу текущего игрока.
4. Устанавливаем `gameActive = false` и выводим сообщение о победителе.

### Алгоритм определения ничьи

1. Если победы нет, проверяем, есть ли `board` хотя бы одна пустая клетка.
2. Если пустых клеток нет - ничья.
3. Устанавливаем `gameActive = false` и выводим сообщение `Ничья!`.

### Алгоритм смены игрока

- Если после хода нет победы и ничьи:
    - `currentPlayer` меняется c `X` на `0` или наоборот.
    - обновляется текст статуса.
